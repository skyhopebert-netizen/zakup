# Стойка — приложение для заказов бариста

Веб-приложение для отправки заявок на закупку поставщикам кофейной стойки.

## Технологии

- React (через CDN, без сборки)
- Supabase — общая база данных для всей команды (заказы, каталог, история)
- localStorage — только для личного имени бариста на конкретном устройстве

## Структура

- `index.html` — точка входа, подключает React/Babel/Supabase и рендерит приложение
- `BaristaOrders.jsx` — основной компонент приложения

## Как это работает

Все данные (каталог товаров, история заказов, пин-код статистики, черновик корзины)
хранятся в общей таблице `app_storage` в Supabase — поэтому все бариста видят
одну и ту же историю и статистику, независимо от устройства.

Имя бариста хранится локально в браузере каждого устройства (`localStorage`) —
оно персональное и не передаётся в общую базу.

## Публикация на GitHub Pages

1. Settings → Pages → Source: `Deploy from a branch`
2. Branch: `main`, folder: `/ (root)`
3. Save — через 1-2 минуты сайт будет доступен по адресу
   `https://<username>.github.io/<repo-name>/`

## Supabase

Таблица `app_storage` создаётся SQL-скриптом `setup.sql`.
Ключ API (`anon public key`) встроен в `index.html` — это нормально,
он специально публичный для фронтенд-приложений и защищён политиками
Row Level Security на стороне Supabase.
